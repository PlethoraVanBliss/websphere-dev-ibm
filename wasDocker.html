Let's run IBM WebSphere Application Server(WAS) for Developers in a Docker Container. The idea here in this post is to create and
save a docker container with WebSphere installed so you can reuse the container whenever you need to run a new WAS in development.
Please check the licensing requirements to run WebSphere at the download page. The steps here should be easily ported and reproduced
to create an image of WebSphere N.D full profile as well if you want.

I will run this steps in a Linux Fedora 20 distribuition but it should be easily ported to other O.Ss.

I will use and run a local image that already has Installation Manager installed as the base initial image,
this image was created from a CentOS image from the public Docker repository. You may want to run your container in a supported linux
version. To do that you will need to create your own Red Hat, Suze or other supported O.S image as it requires a license to use and
won't be available at the Docker public repository.

To follow the steps in this post you will need the following software:

Latest IBM WebSphere Application Server for Developers. -  Currently the latest version is 8.5.5(3 parts) and I am running it using Java 7(3 parts) so you will need to download all files that compose WebSphere + JDK 7 as shown below:

Latest IBM Installation Manager - Currently the latest version is 1.8 as shown below

 

 

Once you downloaded all files create a folder at your prefered workspace location to host the Dockerfile and all downloaded and required files and resources needed to install WebSphere.

The WebSphere container image can't be made available at the docker public repository as Websphere is not Open Source of free software,
but you can share the image at a local registry internally at your company if the company has WebSphere licensed.
